# TAG & NODE BLE 室內定位系統 — 實作規劃（已確認）

## 系統概述

基於 MQTT（Mosquitto）的 BLE 室內定位系統。NODE 是固定錨點，TAG 是移動裝置。
當 TAG 進入 NODE 感應範圍，NODE 自動發佈 `eztag/{CardNo}` 訊息，
包含偵測到 TAG 的 RSSI 值，Server 收集各 NODE 的 RSSI 後，
用**加權質心算法（WCL）**計算 TAG 在平面圖上的座標。

---

## MQTT 訊息格式（已確認）

### NODE 心跳訊息
```
Topic: eznode/{NODE_ID}  例如：eznode/1201A2

Payload 欄位說明：
  CMD    = 命令碼
  ID     = Node ID（與 topic 相同）
  R      = 保留區
  BAT    = 電池電量（HEX）0x64 = 100%
  ST     = 圍籬 flag
  GP     = Group
  CNT    = 封包計數器
  CS     = Checksum
```

### TAG 偵測訊息（定位核心）
```
Topic: eztag/{CardNo}  例如：eztag/1200A3

Payload 欄位說明：
  ID      = 偵測到此 TAG 的 NODE ID  ← 重要
  CardNo  = TAG 卡號（被追蹤裝置）
  R       = 保留區
  BAT     = TAG 電池（HEX）0x4A = 74%
  ST      = TAG 狀態
  GP      = Group
  RSSI    = 信號強度（HEX 有符號 8-bit）← 定位依據
  CNT     = 封包計數器
  CS      = Checksum
```

## 關鍵數值解析

### RSSI（HEX 有符號 8-bit → dBm）
```php
$rssi_hex  = "C1";
$rssi_uint = hexdec($rssi_hex);        // 193
$rssi_dbm  = $rssi_uint > 127 ? $rssi_uint - 256 : $rssi_uint;
// 193 - 256 = -63 dBm（合理值）
```

### 電池電量（HEX → %）
```php
$bat_pct = hexdec("64");  // 0x64 = 100%
$bat_pct = hexdec("4A");  // 0x4A = 74%
```

### RSSI → 距離
```php
// d = 10 ^ ((TxPower - RSSI) / (10 × n))
// TxPower = -59 dBm, n = 2.0（室內）
$distance = pow(10, (-59 - $rssi_dbm) / 20.0);
```

## 定位流程

```
1. MQTT: eztag/1200A3
   ID=120198(NODE), CardNo=1200A3(TAG), RSSI=C1(-63dBm)

2. mqtt_daemon.php 解析 → 寫入 rssi_log

3. 取最近 5 秒同 TAG 所有 NODE 最新 RSSI

4. 查 nodes 表取各 NODE (floor_id, x, y)

5. 加權質心：
   weight_i = 1 / distance_i²
   X = Σ(weight_i × x_i) / Σ(weight_i)
   Y = Σ(weight_i × y_i) / Σ(weight_i)

6. 寫入 tag_positions（REPLACE INTO 覆蓋最新）

7. 前端每 2 秒 AJAX 拉取更新地圖
```
